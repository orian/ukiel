# Ukiel V1 Roadmap

Spec: `docs/superpowers/specs/2026-07-05-ukiel-design.md`. V1 is split into sequential plans, executed in order (rows 1‚Äì7 and 10 executed so far). Each plan produces working, tested software. Detailed plans are written just-in-time (after the previous plan merges) so interfaces referenced are real, not guessed.

**Recommended execution order for the remaining plans: 9 ‚Üí 11 ‚Üí 17 ‚Üí 18 ‚Üí 12 ‚Üí 14 ‚Üí 13 ‚Üí 15 ‚Üí 16 ‚Üí 8.** Rationale: 9 is independent and closes the GC orphan-discovery gap the monitoring spec assumes; 11 builds the perf harness/baseline before 17 changes the file population; 17 rewrites the compactor write path that 12/14/15/16 build on (landing it first avoids mutual adaptation and re-benchmarking); 18's backpressure guards the fleet while the perf plans churn; 8 is written last because pipelines replace the ingest code 18 lands in.
