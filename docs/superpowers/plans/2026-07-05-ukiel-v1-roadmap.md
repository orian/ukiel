# Ukiel V1 Roadmap

Spec: `docs/superpowers/specs/2026-07-05-ukiel-design.md`. V1 is split into sequential plans, executed in order (rows 1–7, 9, 10, and 19 executed so far). Each plan produces working, tested software. Detailed plans are written just-in-time (after the previous plan merges) so interfaces referenced are real, not guessed.

**Recommended execution order for the remaining plans: 20 → 11 → 17 → 18 → 12 → 14 → 13 → 15 → 16 → 8.** Rationale: 20 (metrics P1, write just-in-time) gives the gauges that empirically check everything after it; 11 builds the perf harness/baseline before 17 changes the file population; 17 rewrites the compactor write path that 12/14/15/16 build on (landing it first avoids mutual adaptation and re-benchmarking); 18's backpressure guards the fleet while the perf plans churn; 8 is written last because pipelines replace the ingest code 18 and 19 land in. Per-row notes below.
