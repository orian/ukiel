# Metrics P2: the periodic collector (sketch, roadmap row 21)

Status: **executed** (roadmap row 21). Plan:
`docs/superpowers/plans/2026-07-08-ukiel-metrics-p2.md`; implemented in
`crates/ukield/src/collector.rs` (+ catalog probes in `ukiel-catalog`). Two
deliberate deviations from this sketch landed: the standalone listener binds
only when `metrics_listen` is explicitly set (an unset query-role-less process
warns rather than auto-opening a port), and `ingest_flush_partitions` was
found already shipping from the flusher (P1), so only
`ingest_backpressure_deferrals_total` was added. The metric inventory lives in
`docs/superpowers/specs/2026-07-06-ukiel-monitoring.md`; this note resolved the
*how*.

## Shape

One **collector task** in `ukield`, spawned wherever the recorder is
installed (role-independent), ticking every `collector_interval_ms`
(default 30s, shutdown-token cancelled). Each gauge family is one cheap
indexed query or API call; a failing family logs + increments
`collector_errors_total{family}` and **never** crashes the process — a
monitoring outage must not become a data-plane outage.

## Gauge families and their sources

- **Consumer lag** — `fetch_watermarks` per assigned partition from a
  lightweight metadata-only Kafka client (created from the ingest broker
  config), minus the catalog's `ingest_offsets.next_offset`. Labels
  `{topic, partition}` (bounded).
- **Feed lag** — one SQL join: latest commit id per hypertable minus
  `worker_cursors.last_commit_id`, per `{worker, hypertable}`. Load-bearing
  twice (consumer health + GC reaper fence). Once row 23 (feed horizon)
  lands, also emit `feed_horizon_margin_seconds` — distance between each
  cursor and the prune horizon; this becomes the horizon's safety alarm.
- **Live parts** — `GROUP BY hypertable_id, level` count → 
  `catalog_live_parts{hypertable, level}`; after plans 17/18, add backlog
  runs per level (the `finalize_candidates`-style aggregate) and
  `compactor_unfinalized_partitions` / oldest-unfinalized age.
- **GC backlogs** — `pending_objects` count + oldest age, unpurged
  tombstone count, reap-fence age: four indexed scalars.
- **Pool** — `sqlx` pool `size()` / `num_idle()` directly.
- **Cache dir** — free-space percent via statvfs every tick; cache
  directory byte total via a bounded walk every Nth tick (walks are the
  expensive one).

## Standalone listener (closes the P1 role-split gap)

Config `metrics_listen` (default unset). When set — or when the query role
is absent — serve `/metrics` + `/healthz` on a minimal axum router bound to
it. The P1 `OnceLock` handle is reused; nothing else changes.

## Constraints carried forward

Label discipline unchanged (never namespace). All queries must hit existing
partial indexes; if a family needs a new index, the plan adds it explicitly
rather than letting a gauge become a seq scan at 10⁶ parts.
